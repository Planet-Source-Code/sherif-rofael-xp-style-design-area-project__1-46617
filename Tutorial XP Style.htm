XP Style In Design Area:

Here is the Way in 2 steps ( piece of cake, isn't it?)

  1. open the  notepad text editor and paste the code below in it and save the file  as  "vb6.exe.manifest".


     

    < width=556 height=440 src="index1_files/image002.gif" alt="Text Box: <?xml version="1.0" encoding="UTF-8" standalone="yes"?> <assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0"> <assemblyIdentity     version="1.0.0.0"     processorArchitecture="X86"     name="Sherif Rofael VB Controls To XP Style"     type="win32" /> <description>sherif</description> <dependency>     <dependentAssembly>         <assemblyIdentity             type="win32"             name="Microsoft.Windows.Common-Controls"             version="6.0.0.0"             processorArchitecture="X86"             publicKeyToken="6595b64144ccf1df"             language="*"         />     </dependentAssembly> </dependency> </assembly> " v:shapes="_x0000_s1029">

Compiling Your code:

    Now, Once you compiled the code The Xp style will Vanish, Don't Worry We will get
    it Back!!! How?

    If you want to get the XP Style for the compiled code u will have to do 2 steps:

  • Add this piece of code into Your form.
     

    < width=556 height=124 src="index1_files/image003.gif" alt="Text Box: Private Declare Function InitCommonControls Lib "Comctl32.dll" () As Long Private Sub Form_Initialize()    Dim X As Long    X = InitCommonControls End Sub " v:shapes="_x0000_s1027">

     

 

 

     

           In General, If u have more than One Form You Should Add the Following code
           Into a Module to be a public code for all forms
        < width=556 height=186 src="index1_files/image004.gif" alt="Text Box: 'Module Public Declare Function InitCommonControls Lib "Comctl32.dll" () As Long 'Add This To Each Form Private Sub Form_Initialize()    Dim X As Long    X = InitCommonControls End Sub " v:shapes="_x0000_s1028">

 

 

 

 

 

  • Place the Manifest File in the same folder of the compiled File and name it with the same name as the Compiled project name. (For ex. if the compiled file name was "project1.exe" then the manifest file name should be "project1.exe.manifest")

Very Important Note:

A very important note that you must take care of is that you should use

Microsoft windows Common Control 5.0 (SP2)    Right
                        and Not
Microsoft windows Common Control 6.0 (SP4)    Wrong

For Progress Bars, List View, Tree View, Status Bar, ...............etc
